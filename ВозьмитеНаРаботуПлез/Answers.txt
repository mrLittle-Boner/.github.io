Q1
Расскажите, чем, на ваш взгляд, отличается хорошая верстка от плохой с точки зрения

 -пользователя : для пользователя самое главное удобство расположенных элементов и понятность
	хорошая вёрстка - адаптивная под устройство и интуитивно понятная(куда кликать для достижения цели)
	плохая - не адаптивная и интуитивно непонятная.

 -менеджера проекта : предположу, что для PM`а главное простая и понятная вёрстка с дальнейшим расширением в кратчайшие сроки
	хорошая вёрстка - хорошо написана и легко расширяется
	плохая - наоборот

 -дизайнера: возможно, для дизайнера хорошая вёрстка эта та в которую можно добавить или изменить пару элементов без потери всего контекста
	хороая вёрстка - дополнаяется и меняются некоторые элементы без потери общего смысла
	плохая - сжатая и однонаправленная по дизайну

 -верстальщика: для верстальщика, наверное, хорошая вёрстка заключается в понятности написанных элементов и классов , быстрой ориентации по вёрстке
	хорошая вёрстка - элементы свёрстаны правильно и имена классов понятны для дальнейшей работы с вёрсткой
	плохая - нарушение правил вёрстки, классы с непонятными названиями

 -клиентского програмиста: если речь идёт о frontend, то тут ,скорее всего, хорошим будет наполнение врёстки элементов для работы с ними через JS
	хорошая вёрстка - есть элементы с классами для дальнейшей работы с ними через javascript
	плохая - нет классов у нужных элементов, возможно неоптимизированные картинки

 -серверного програмиста: не сильно представляю как серверный программист связан с вёрсткой, но попробую
	хорошая вёрстка - наполнена формами и всяческими элементами для вывода информации с сервера
	плохая - без форм и специальных элементов на страницы

Q3
Опишите основные особенности верстки сайтов, которые должны одинаково
хорошо отображаться как на любом современном компьютере, так и на
смартфонах и планшетах под управлением iOS и Android. Расскажите о своем
опыте верстки подобных сайтов: какие инструменты и технологии вы применяли,
как проверяли результат на различных устройствах, какие именно устройства
требовалось поддерживать.

Ответ: Для корректного отоброжения сайта на разных устройствах стоит сверстать адаптивно или "резиново",
так сайт будет подстраиваться под разрешение экрана, так же, для приятного отображения на мобильных устройствах,
макет сайта может быть сделан под мобильные устройства(Mobile first).
Для достижения отображения сайта на разных устроийствах в стилях нужно прописать медиа запросы ,
которые будут применять разные стили в соотвествии с заданными параметрами.
Чтобы сайт отоброжался правильно на разных устройствах надо применять технологии которые есть в обоих платформах,
или по другому - кроссплатформено и кроссбраузерно, для этих целей может пригодиться ресурс - caniuse - на котором можно проверить совместимость
какой либо технологии с браузером или системой. Так же для устройств на базе iOS с экранами типа Retina следует использовать картинки с большей 
плотностью пиксилей.На данный момент я не использовал макеты Mobile first а обычные адаптивные сайти проверял через DevTools браузера
в котором можно переключиться на проверку сайта с помощью эмуляции разной шерены экрана. Для достижения адаптивности я использовал обычные меда запросы.
